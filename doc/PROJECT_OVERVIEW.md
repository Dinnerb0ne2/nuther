# Nuther 项目概述

## 项目简介

Nuther (Retro Memory LSTM) 是一个基于 LSTM + 记忆回溯算法 + MoE 混合专家机制的对话式神经网络框架。该项目不使用 GPU 加速，完全使用 NumPy 实现，旨在实现一个能够进行知识问答和交流的对话式神经网络模型。

## 核心特性

### 1. LSTM 核心引擎
- 纯 NumPy 实现的 LSTM 网络
- 支持多层堆叠和双向处理
- 完整的门控机制（遗忘门、输入门、输出门）
- 支持带嵌入层的文本处理

### 2. 记忆回溯机制
- 基于关键词加权的相似度计算
- 支持分块存储和检索
- LRU 淘汰策略管理内存
- 自动提取关键词，提高检索效率

### 3. MoE 混合专家
- 稀疏路由机制（Top-K 选择）
- 多种门控策略（Top-K、Soft、Gumbel-Softmax）
- 负载均衡防止专家坍缩
- 支持多种专家类型（前馈网络、LSTM）

### 4. 对话式交互
- 多轮对话支持
- 自动保存对话历史
- 上下文增强的响应生成
- 会话管理和统计

### 5. 知识库管理
- 网页爬虫功能
- 文本清洗和预处理
- 知识分块存储
- 高效的相似度检索

## 技术架构

### 整体架构图

```
用户输入
    ↓
[输入编码] → 词汇表 → 索引转换
    ↓
[编码器] → EmbeddingLSTM → 记忆检索 → 上下文增强
    ↓
[解码器] → LSTM + MoE → 输出投影
    ↓
[输出解码] → 索引转换 → 文本生成
    ↓
用户输出
```

### 数据流

1. **输入处理流程**
   - 文本输入 → 分词 → 词汇表索引 → 嵌入层 → LSTM 编码

2. **记忆检索流程**
   - 编码输出 → 相似度计算 → Top-K 检索 → 上下文融合

3. **输出生成流程**
   - 上下文增强 → LSTM 解码 → MoE 投影 → 词汇表解码 → 文本输出

## 模块组成

### 核心模块

1. **配置模块** (`src/config/`)
   - 管理所有超参数和配置
   - 模型架构参数
   - 训练参数
   - 记忆和 MoE 参数

2. **词典模块** (`src/vocab/`)
   - 文本到索引的转换
   - 词典构建和管理
   - 特殊令牌处理
   - 批量编码/解码

3. **数据处理模块** (`src/data/`)
   - 对话序列构建
   - 序列填充和归一化
   - 数据加载器
   - 批处理支持

4. **爬虫模块** (`src/crawler/`)
   - 网页内容爬取
   - 文本清洗和预处理
   - 知识库存储
   - 示例知识库生成

5. **LSTM 模块** (`src/lstm/`)
   - LSTM 单元实现
   - LSTM 层封装
   - 堆叠 LSTM 网络
   - 嵌入 LSTM

6. **记忆模块** (`src/memory/`)
   - 记忆块管理
   - 记忆存储系统
   - 相似度计算
   - 记忆检索器
   - 记忆库整合

7. **MoE 模块** (`src/moe/`)
   - 专家模型实现
   - 门控网络
   - MoE 整合
   - 稀疏 MoE

8. **模型模块** (`src/model/`)
   - 编码器实现
   - 解码器实现
   - Nuther 模型整合

9. **对话模块** (`src/chat/`)
   - 会话管理
   - 聊天机器人
   - 交互式对话

10. **主程序** (`src/main.py`)
    - 框架管理器
    - 命令行接口
    - 模块协调

## 设计理念

### 1. 模块化设计
每个模块都有明确的职责和接口，便于独立开发和测试。

### 2. 可扩展性
支持多种专家类型、门控策略和记忆检索方式，易于扩展新功能。

### 3. 纯 NumPy 实现
不依赖深度学习框架，使用纯 NumPy 实现，便于理解和学习。

### 4. 记忆增强
通过记忆回溯机制，模型能够利用历史信息进行推理和回答。

### 5. 专家专业化
通过 MoE 机制，模型能够根据输入选择合适的专家进行推理。
